Requirements

  1. Exercises must be quick to do (1-2 minutes)
    • People learn best in small steps
    • People drop out if exercises are longer
  2. Answers must be automatically gradable
    • Human inspection doesn't scale affordably
    • Instant feedback is more engaging

Multiple Choice Question

  • Short, unambiguous question
  • 3-5 responses
    • Remember 7±2
  • Exericse reasoning and judgment, not just recall

Multiple Choice Question (Example)

What does the following fragment of Python print?

letters = 'abc'
for char in letters:
    print(char)
    letters.upper()
  1. A B C
  2. a B C
  3. a b c
  4. error
  • Remember: wrong answers should be chosen to diagnose misconceptions

Write and Run

  • Write code that does X

The matrix M contains data read from a file. Using one function or method call, create a matrix Z that has the same shape as M but contains only zeroes.

Write and Run (Improved)

  • Blank screens can be intimidating
  • So have learners fill in the blanks so that the code does X
  • Easier to auto-grade answers if you provide a starting point

Fill in the blanks so that the code below prints the string 'hat'. 

text = 'all that it is'
slice = text[____:____]
print(slice)

Minimal Changes

  • Modify code instead of filling in blanks
  • Change the output
  • Or fix the bug

Minimal Changes (Example)

This function is supposed to test whether a point (x, y) lies strictly within a rectangle defined by (x_min, y_min, x_max, y_max). Change one line to make it do so correctly. 

def inside(point, rect):
    if (point.x <= rect.x_min): return false
    if (point.y <= rect.y_min): return false
    if (point.x >= rect.y_max): return false
    if (point.y >= rect.y_max): return false
    return true

Minimal Changes (Continued)

  • Change one variable's initial value
  • Swap an inner and outer loop
  • Change the order of tests in a chain of conditionals
  • Change the nesting of function calls

Change the inner loop control in the function below so that it sets the upper left triangle of the matrix to zero. 

def zeroTriangle(matrix):
    for c in range(matrix.cols):
        for r in range(matrix.rows):
            matrix[r, c] = 0

Writing Tests

  • Novices will spend most of their time debugging
    • We're all novices, really…
  • Give students a chance to practice another real-world skill
  • Exercise higher-level thinking
    • Reason from effect back to cause

Writing Tests (Example)

monotonic_sum calculates the sum of each section of a list of numbers in which the values are monotonically increasing. For example, given the input [1, 3, 3, 4, 5, 1], the output should be [4, 3, 9, 1]. Which of these bugs does it contain?

  1. Every negative number starts a new sub-sequence.
  2. First value of each sub-sequence not in sub-sum.
  3. Last value of each sub-sequence not in sub-sum.
  4. Only re-starts sum when values actually decrease.

Parsons Problem

  • Give learner the lines of code in jumbled order
  • Require them to put them in order to produce desired effect
  • Reduces cognitive load by allowing them to focus on sequence rather than vocabulary
  • Giving the student more lines than needed or requiring them to add lines makes problem significantly harder

Parsons Problem (Example)

Rearrange and indent these lines to calculate the sums of the positive and negative values in a list. 

positive = 0
return negative, positive
if v > 0
else
positive += v
negative = 0
for v in values
negative += v

Tracing Execution

  • Given code, trace order in which lines are executed
    • Inverse of a Parsons Problem
  • Authentic task (comes up frequently in debugging)
  • Not supported, but can be emulated with MCQ

Tracing Execution (Example)

In what order do the lines in this code execute? 

A)     vals = [-1, 0, 1]
B)     inverse_sum = 0
       try:
            for v in vals:
C)             inverse_sum += 1/v
       except:
D)         pass
  1. A, B, C, C, C
  2. A, B, C, C, D
  3. A, B, C, C, D, C

  • Can also trace successive values

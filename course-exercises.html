Requirements

  1. Individual exercises must be short (1-2 minutes)
    • People learn best in small steps
    • People drop out if exercises are longer
  2. Answers must be automatically gradable
    • Instant feedback is more engaging
    • Human inspection doesn't scale affordably

Multiple Choice Question

  • Short, unambiguous question
  • 3-5 responses
    • Remember 7Â±2
  • Exercise reasoning and judgment, not just recall

Multiple Choice Question (Example)

What does the following fragment of Python print?

letters = 'abc'
for char in letters:
    print(char)
    letters.upper()
  1. A B C (doesn't understand instruction sequencing)
  2. a B C (thinks upper mutates its object)
  3. a b c (correct answer)
  4. error (thinks method results have to be assigned)

Minimal Changes

  • Modify code instead of filling in blanks

Fix function so that it tests whether a point (x, y) lies strictly within a rectangle defined by (x_min, y_min, x_max, y_max). 

def inside(point, rect):
  if (point.x <= rect.x_min): return false
  if (point.y <= rect.y_min): return false
  if (point.x >= rect.y_max): return false
  if (point.y >= rect.y_max): return false
  return true

Minimal Changes (continued)

  • Change one variable's initial value
  • Swap an inner and outer loop
  • Change the order of tests in a chain of conditionals
  • Change the nesting of function calls
  • Debugging

Finding Bugs

  • Novices will spend most of their time doing this
    • Just like the rest of us
  • Give students a chance to practice another real-world skill
  • Exercise higher-level thinking
    • Reason from effect back to cause

Writing Tests (Example)

subsum sums each sub-list of strictly inreasing numbers: subsum([1, 3, 3, 4, 5, 1]) produces [4, 3, 9, 1]. Which bug does it contain?

  1. Every negative number starts a new sub-sequence.
  2. First value of each sub-sequence not in sub-sum.
  3. Last value of each sub-sequence not in sub-sum.
  4. Only re-starts sum when values actually decrease.

Tracing Execution

  • Given code, trace order in which lines are executed
  • Authentic task (comes up frequently in debugging)
  • Can be emulated using multiple choice question
    • Though this does increase cognitive load
  • Can also trace successive values

Tracing Execution (Example)

In what order do the lines in this code execute? 

A)     vals = [-1, 0, 1]
B)     inverse_sum = 0
       try:
            for v in vals:
C)             inverse_sum += 1/v
       except:
D)         pass
  1. A, B, C, C, C
  2. A, B, C, C, D
  3. A, B, C, C, D, C

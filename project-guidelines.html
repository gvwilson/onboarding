Overview

  1. Consistency reduces friction when learning
  2. We want to implement good teaching practices
  3. Our data tells us how to make online material more engaging

Context

  • 30-800 characters (5-130 words)
  • 1-3 paragraphs
  • Inclusive voice
    • We are doing this together
    • "Let's removed Survived from the dataset."
  • Avoid exclamation marks!
    • And remember that most jokes aren't funny the second time around

Instructions

  • Opening paragraph: 60-180 characters (10-30 words)
  • Then 1-4 items totalling at most 600 characters (100 words)
  • Less detailed than in a course
    • Meant to remind rather than instruct

Good Instructions

  • State the objective of the task
    • "Create a dataframe called df containing 50 points on a circle and plot it."
    • Specify variable names to simplify checks
  • Make sub-goals explicit
    • "Create a vector of 50 angles from 0 to 2π radians."
    • "Display a scatterplot using ggplot2."
  • Make acceptance criteria very clear
    • False negatives are extremely frustrating

Starter Code

  • Also less detailed than in a course
  • Use the imperative voice in comments
  • Wrong: 
    # This sets plots to a nice size.
options(repr.plot.width = 4)

# This loads the ggplot2 library.
library(ggplot2)
  • Right: 
    # Customize plot width.
options(repr.plot.width = 4)

# Load ggplot2.
library(ggplot2)

Hints

  • No more than 800 characters (130 words)
    • Preferably much less
  • May include scaffolded code
  • "Assemble the data frame df using
    data.frame(t = t, x = sin(t), ___)"

Solutions

  • 1-10 lines
  • Should run in 5 seconds or less on a regular laptop
  • So add datasets to the Docker image with requirements.sh unless it's completely impractical

What We Just Did

  1. Outlined guidelines for project content
  2. Included enough rationale and examples to make guidelines believable
    • Because that's a good teaching practice

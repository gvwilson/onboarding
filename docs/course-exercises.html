Overview

  • Now understand goals for exercises and constraints on memory and cognition
  • See how different kinds of exercises on our platform achieve that

Constraints

  1. Individual exercises must be short (1-2 minutes)
    • People learn best in small steps
    • People drop out if exercises are longer
  2. Answers must be automatically gradable
    • Instant feedback is more engaging
    • Human inspection doesn't scale affordably

Lessons

  • Basic unit of a course
  • One video followed by 2-4 exercises
  • Video consists of slides and script
  • Exercises consist of:
    • assignment text
    • instructions
    • hints
    • code (possibly several pieces)

Videos

  • Give students an overview of what the exercises will do in steps
  • 2-4 minutes (400-500 words)
  • Minimal content on each slide
    • 3 slides instead of 6 bullets on one slide
    • Favor images and diagrams over bullets
    • And bullets over paragraphs

Text

  • Assignment text before exercise instructions should be short and sharp
    • Only include text and code relevant to the exercise
    • Do not introduce new concepts
  • Instructions should be four or fewer bullets
  • Hint should give the learner half of the solution

Execution Model

  • Pre-exercise code runs silently before the learner is able to run their own code
    • E.g., read data into a data frame
  • Sample code is put in their console for them to start editing
    • Comments labelling steps in solution
    • Partially-completed code (fill in the blanks)
    • Jumbled lines of the solutions (Parsons Problem)
    • Buggy code to be fixed

Execution Model

  • Instructor and Content Developer (CD) write Submission Correctness Tests (SCTs)
    • Code to check code
  • Also write a sample solution to be shown to learners who ask for it
  • When we build and test the course, the sample solution must pass the SCTs
  • When the learner does the course, we check their code with the same SCTs

Pure Multiple Choice Exercise

  • Learner does not write or run code
  • Short, unambiguous question
  • 3-5 choices
    • Remember 7Â±2
    • Strive for diagnostic power
  • Write informative messages for both correct and incorrect responses
  • Exercise reasoning and judgment, not just recall

Multiple Choice Exercise

  • Learner gets an interactive console to experiment in before choosing an answer
  • But we don't check their code or its output, just their selected answer
  • Feedback on incorrect answers should help them figure out what to try next
    • Again, write informative messages for all responses

Normal Exercise

  • Write code that does X
    • But blank screens are intimidating
  • So give learners sample code
    • Also makes auto-grading more reliable

Multi-Step Exercises

  • Tab exercise presents steps that build on each other one at a time
    • Steps displayed one at a time
    • E.g., build ggplot2 visualization bit by bit
  • Bullet exercise presents independent tasks related to the same concept
    • All steps displayed at once
    • So they must be short (1-2 lines)
    • E.g., select film titles, then select authors, then select country and release year

Find and Fix Bugs

  • Another way to use coding exercises of all kinds
  • Novices will spend most of their time doing this
    • Just like the rest of us
  • Exercises higher-level reasoning
    • Reason from effect back to cause
  • Either a normal exercise or a multiple choice exercise

Engagement

Prefer coding exercises over non-coding exercises

  • People come to us for hands-on instruction
    • If they wanted a book, they'd buy a book
  • Applying new knowledge right away improves retention
    • Knowledge is more likely to be remembered
    • People are more likely to complete the course

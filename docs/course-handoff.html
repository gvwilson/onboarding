Instructor Onboarding:
Course Handoff

Version 0.1

See http://authoring.datacamp.com/ for full documentation

Overview

  • Design the course with a Curriculum Lead (CL)
  • Then work with a Content Developer (CD) on:
    • Production
    • Deployment
    • Maintenance
  • Process designed to maximize productivity and minimize surprises

Handoff Process

  • Instructor, CL, and CD go through the design together
    • Fresh eyes inspire fresh questions
  • Define a clear timeline with short-term and long-term targets
  • Ensure the instructor has all necessary documentation

Workflow After Handoff

  • Weekly 30 minute check-ins
  • Allow the CD at least one full day to review content changes
  • Develop chapters in order
    • A hard requirement
  • Do not move past Chapter 1 until the CD approves it

Mechanics

  • Work in the develop branch
    • CD will deploy to master when the course is ready for learners
  • Use GitHub pull requests to request feedback
    • Allows line-by-line review
    • And threaded discussion with many participants
  • Make sample and solution code identical at the start
    • Sample code can then be faded out based on exercise instructions

Soft Launch

  • DataCamp arranges this before general availability
  • Ask existing customers to try the course and give feedback
    • Typically takes about a week
    • Always finds something
  • Best for everyone to address this feedback as quickly as possible
    • If you're going to take a couple of days to focus on your course, this is the time to do it

Launch

  • DataCamp will announce new courses to all existing users
  • You should also tweet, blog, post to mailing lists, etc.

Checkpoint #1

  • Two weeks after launch or after 100 completions
  • Goal is to resolve critical unexpected issues
    • No major content changes
  • CD will assist in reviewing data and handles adjustments to SCTs
  • Instructor does content changes

Checkpoint #2

  • Six weeks after launch (i.e., one month after checkpoint 1
  • Initial traffic spike will have dissipated
  • Completion rate at this point is (probably) a stable indicator
    • This is the point to think about changes to ordering

Checkpoint #3

  • Six months after launch
  • Re-examine the course:
    • Are the packages used up to date?
    • Are the methods taught still relevant?
    • Are there better ways to teach core concepts?

Maintenance

  • Instructor is responsible for all content not related to SCTs
  • Use Review Mode in Teach
    • Provides qualitative as well as quantitative data
  • Most important metrics are:
    • Completion rates at the exercise, chapter, and course level
    • Percentage of students asking for a hint or solution in a given exercise

We're Here to Help

  • Provide hands on assistance during Checkpoints 1 and 2
    • I.e., two and six weeks post course-launch
  • Demonstrate how to access and interpret course metrics
  • Improve SCTs based on data
  • Flag course metrics that are becoming concerning

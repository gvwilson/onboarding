Memory and Cognition

Overview

  • A novice doesn't (yet) have a mental model of the problem domain
  • A competent practitioner can succeed at normal tasks under normal circumstances with normal effort
    • For example, bike to work or make dinner
  • Look at what distinguishes an expert in order to see how to design lessons that work for people at all levels

Characteristics of Experts

  • Solve common problems "instantly"
  • Able to solve unusual problems as well
  • Able to reason backward from effect to cause

But:

  • Sometimes unable to explain their reasoning
  • Often mis-use words like "simple" or "obvious"

A Mental Model of Expertise

  • Knowledge isn't stored as a graph, but it's a useful model
  • Expert's graph is much more densely connected
  • So experts can reach solutions in one step instead of many
    • With much less backtracking

You Are Not Your Learners

  • Experts recognize solutions instead of constructing them
  • Can't explain their method because there wasn't one
  • The best teacher is someone who mastered the topic recently
    • Because they still remember what it's like to not understand
  • Explicitly itemizing required steps helps overcome expert blind spot

Concept Maps

  • A picture of a mental model

  • Emphasize connections just as much as concepts
  • Knowledge that cannot be retrieved cannot be used

Human Memory Hierarchy

  • Sensory memory: 1-2 seconds
  • Short-term memory: 7±2 items for 10-15 seconds
  • Long-term memory: permanent storage
    • But frequently rewritten
  • Learning is the act of transferring new information to long-term memory
  • But new information initially goes into short-term memory
  • If we give learners too much too fast, it spills out of short-term memory and learning doesn't occur

Lesson Size

  • Count the nodes and edges in the concept map
    • Or keywords in a point-form outline that enumerates connections
  • Each learning episode should be at most half a dozen items

Reinforcement

  • Transference is more likely if new information is reinforced to keep it in short-term memory
  • Good: explain it again in a different way
  • Better: have learners use it
    • Which also checks that they understood it the first time
  • Music and athletics: instructor and learner alternate every 30-60 seconds
  • Try to achieve this with code

Retrieval

  • "Memory is the residue of thought." - Willingham
  • No point storing information if we can't retrieve it
  • Retrieval improves with practice
    • Quizzes and flash cards significantly outperform repeated study
  • (Mostly) domain-specific
    • Little evidence for "far transference"
    • I.e., improvements in one domain don't generally produce improvements in others

Cognitive Load

  • Different parts of problem solving compete for processing time (e.g., recall vs. planning)
  • Good lessons let learners concentrate on one aspect of a problem at a time
  • Parsons Problems
    • Un-jumbling lines of code to solve a problem
    • Allows learner to concentrate on data flow instead of vocabulary
  • Not yet directly supported in DataCamp courses, but easy to emulate

Faded Examples

  • A succession of closely-related problems
  • Each gives the learner less scaffolding
  • Parts that are removed are closely related
  • Learners focus on one aspect of the problem at a time
    • And learn the underlying pattern by example

Chunking

  • Items that frequently occur together are stored in chunks that only take up one slot in short-term memory
    • The digits in frequently-used phone numbers
    • Common chord progressions
  • Chunking lets us work with larger information sets
    • At the risk of mis-categorizing items
  • Good lessons give chunks names
    • "Flag"
    • "Best value seen so far"

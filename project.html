Overview

  • Allow learners to apply what they've learned…
  • …to an end-to-end analysis on a real-world task…
  • …and showcase their work
  • A summative assessment
    • Or self-assessment

Architecture

  • A Git repository containing:
    • A Jupyter Notebook with specially-formatted cells
    • Metadata in a separate file
    • Datasets, images, etc.
    • Requirements script to install packages, download datasets, etc.
  • Edit the notebook directly or edit and preview online
  • Work in the develop branch
    • master is used to deploy
    • DataCamp will merge your work into it

The Notebook

  • 6-8 steps
  • Each step provides context
  • Then instructions specifying what to do
  • And also:
    • Title
    • Sample code (starting point for user's work)
    • Hint (show if asked)
    • Tests (for the user's code)
    • Solution (check the tests and show if asked)

Voice

  • Context is written like a blog post
    • "Suppose we want to…"
    • "Next, we load the libraries…"
  • Instructions are written as direct imperatives
    • "Load the library."
    • "Plot the expected distribution."
  • Be brief
    • Users shouldn't have to scroll within a step

Behind the Scenes

  • We use Docker to provide a fresh Linux virtual machine for each project
  • Customize image by writing a requirements.sh script to be executed when image built
    • Install packages and datasets, etc.

Tests

  • Tests use the standard nose library
    • Impractical to test everything users might do
    • Projects are much less constrained than course exercises
  • Test data structures rather than generated figures
  • Users can run individual cells as they go along
    • Or select the next heading and "run above" to check cumulative effect
  • Users can add cells (but probably shouldn't)

Metadata

  • Stored in project.yml
title Project title
description Brief description
long_description Full description
language python, r, etc.
instructors ID(s) of course author(s)
collaborators ID(s) of other people involved
time_needed E.g., "45 minutes"
…and other fields that DataCamp will fill in

Project Specification

  • First step is to fill in a project spec (also a Notebook)
  • Name, email, short and long descriptions, datasets, student background
  • First three tasks of the project
    • Without instructions, hint, or tests
    • But with solution
    • What you would put in a blog post
  • All of this is re-used in the completed project

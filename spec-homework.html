Overview

  • Five short exercises to reinforce what you have just learned and spark discussion
  • Please do before coming to the live discussion
  • Share by submitting a PR to your session's GitHub repository

1. Multiple Choice Question

Create a multiple-choice question that tests understanding of one of the concepts you intend to cover in your course. Include the correct answer and 2-3 incorrect answers, along with an explanation of the misconception each incorrect answer is intended to diagnose.

2. Debugging Exercise

Write a brief description of a debugging problem you might have learners solve during your course. Include a description of what the code is supposed to do, and 5-10 lines of code that almost do that.

3. Concepts

Create a point-form list of the concepts and relationships that are used in the debugging exercise you just created. (This is a textual substitute for a concept map.)

4. Stumbling Blocks

Think about the language features and libraries your learners will use, and the terminology they will encounter. What do you think will be most confusing or hardest to justify? (For example, it's often hard to explain to newcomers why Python has both lists and tuples.)

5. Personal Motivation

Create a point-form summary of what you hope to get out of developing a course for DataCamp.
